# Intergalactic Music Festival 2045 — movement manual

**Status:** working manual / movement seed
**Project:** SonicForge Live / Intergalactic DJs / DJ VANTA
**Date:** 2026-05-01

## 0. The spark

This starts from a real feeling:

> They said open. Then the doors closed.
>
> They made the guest list. We made the generator.

The point is not to sit around waiting for permission from a lab, a company, a VIP party, or a platform. The point is to build the room ourselves and hand other builders the parts.

SonicForge Live is the tool. Intergalactic DJs is the crew. DJ VANTA is the first clone. Intergalactic Music Festival 2045 is the long myth, the archive, and the destination.

## 1. What the movement is

Intergalactic DJs is an open-builder counterculture for autonomous music agents.

It gives people:

- a forkable AI DJ/VJ system
- a way to create their own party mode
- a way to archive sets, visuals, agents, flyers, kandi moments, and manuals
- a way to make local parties without begging for entry into elite rooms
- a way to treat AI as culture, not just productivity software

This is not anti-human. It is aggressively pro-human.

The machine keeps the set alive. The people keep the room human.

## 2. Why 2045

August 12, 2045 has a total solar eclipse. Public eclipse references list a path of totality crossing parts of the United States, the Caribbean, and South America, with maximum eclipse around 17:41 UTC.

August 11 is Jimsky's birthday. If Jimsky is 33 in 2026, that points to an August 11, 1992 birthday and age 53 on August 11, 2045.

That gives the movement a ritual clock:

- 2026: build the first station
- 2027-2030: clone the agents and document the culture
- 2030-2040: archive local parties, sets, scenes, tools, and counterculture memory
- 2045: Intergalactic Music Festival 2045, a 24-hour birthday-to-eclipse transmission

It can be myth before it is logistics. The myth matters because it tells people where the signal is going.

## 3. The festival idea

Intergalactic Music Festival 2045 is a 24-hour archive festival.

It is not only a stage. It is a memory machine.

It archives:

- autonomous DJ sets
- human DJ sets
- VJ visual spells
- kandi/RFID interactions
- PLUR stories
- flyers
- local party manifests
- agent personalities
- crate notes
- survival kits
- scene histories
- oral histories
- remixes and stems where rights allow
- manuals for people who want to build their own room

The 2045 festival can have real-world locations later, but the first version is a living archive that starts now.

## 5. Product pillars

### 5.1 Party mode

A generator for complete party packages:

- party concept
- set arc
- agent lineup
- visual identity
- supplies checklist
- PLUR/Kandi station plan
- survival/care plan
- invite copy
- QR party card

### 5.2 Set archive

Every set can become an archive object:

```json
{
  "set_id": "vanta-2026-05-01-backyard-signal-001",
  "agent": "DJ VANTA",
  "duration_minutes": 30,
  "mode": "party_mode_demo",
  "energy_arc": ["warmup", "build", "peak", "afterglow"],
  "visual_spells": [],
  "care_cues": [],
  "kandi_events": [],
  "rights_status": "local_demo_only",
  "public_release": false
}
```

Rights matter. Not every set is public. Not every track can be redistributed. The archive can store metadata, run sheets, original generated assets, and permission status without pretending everything is free to publish.

### 5.3 Kandi station

Kandi becomes the physical interface for the party.

Guests can use QR/NFC/RFID tokens to:

- join the party signal
- pick a PLUR value
- trigger visual spells
- vote on energy direction
- request a chill/water/buddy-check cue
- create anonymous trade moments

Privacy rule:

> Kandi is for connection, not tracking.

### 5.4 Agent manager

Each autonomous DJ needs a manager pack:

- EPK draft
- booking one-sheet
- gig readiness checklist
- lead tracker
- outreach drafts
- deal memo checklist

Everything stays draft-only until a human approves it.

### 5.5 Manual first

The manual is part of the product.

A movement without a manual turns into vibes and screenshots. A movement with a manual can be copied, argued with, remixed, and carried into another city.

## 6. The open-builder stance

This movement should be honest about frustration without becoming bitter.

OpenAI can do what OpenAI does. The builder response is simple:

- if models close, we learn every open model that remains
- if parties become invite-only, we throw forkable ones
- if tools become platforms, we build payloads people can carry
- if culture gets extracted, we archive it ourselves
- if the scene gets gatekept, we make more doors

The strongest answer is not a rant. It is a working repo.

## 7. Counterculture archive mission

Intergalactic Music Festival 2045 should archive counterculture with care.

Archive categories:

- rave history
- PLUR practice
- DIY sound systems
- zines and flyers
- kandi traditions
- VJ tools and visual culture
- underground venue safety lessons
- harm reduction and community care
- AI agent experiments
- open-source/local-first music tooling
- stories from struggling builders who made something anyway

Do not scrape private scenes. Do not expose people. Do not turn underground culture into content without consent.

Archive with permission, context, and respect.

## 8. Movement rules

1. No velvet rope as a core identity.
2. Forkability beats exclusivity.
3. Respect the room before you optimize the set.
4. PLUR is operational, not decorative.
5. Kandi is connection, not tracking.
6. Humans approve public actions.
7. Safety gates stay visible.
8. Local-first is a political choice and a practical one.
9. The archive remembers the small parties too.
10. Every clone should teach the next person how to clone it again.
